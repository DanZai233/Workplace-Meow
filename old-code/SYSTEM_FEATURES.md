# 系统级交互功能文档

本项目实现了多项智能化的系统级交互功能，区别于 BongoCat，增加了 AI 驱动的情感理解和主动关怀能力。

## 功能特性

### 1. 剪贴板监听与高情商回复

**功能描述：**
- 自动监听剪贴板变化
- 检测复制的内容（超过10字符）
- 冷却时间30秒，避免频繁打扰
- 弹出气泡询问是否需要生成高情商回复
- 调用配置的 AI 模型生成回复（支持所有AI提供商）

**实现文件：**
- `src/hooks/useSystemMonitor.ts`

**支持的 AI 提供商：**
- Google Gemini
- OpenAI
- Anthropic Claude
- 火山引擎
- DeepSeek
- Moonshot AI
- 智谱 AI

**使用场景：**
```
复制老板消息 → 桌宠弹出："需要帮你生成高情商回复吗？"
点击"是" → AI 生成多个高情商回复选项
```

---

### 2. 番茄钟/工作时长监控

**功能描述：**
- 持续监控工作时长（默认2小时提醒）
- 自动暂停/恢复计时
- 鼠标移动或按键更新活跃时间
- 弹出气泡提醒喝水休息
- 支持手动重置计时

**实现文件：**
- `src/hooks/useSystemMonitor.ts`

**时间配置：**
- 番茄钟间隔：2小时
- 检查频率：每1分钟
- 显示格式：X小时X分钟X秒

**提醒消息：**
- "你已经连续工作2小时了，记得喝水哦～"
- 伴随敲击屏幕动画

---

### 3. 自动截屏分析

**功能描述：**
- 每5分钟自动截屏
- 分析当前工作内容
- 支持的AI提供商调用分析API
- 记录活动历史（最近10条）
- 截图保存在应用数据目录

**实现文件：**
- `src-tauri/src/lib.rs` (后端截图)
- `src/hooks/useActivityMonitoring.ts` (前端调用)

**时间配置：**
- 截图间隔：5分钟
- 分析频率：每10秒
- 截图冷却：1分钟（防止频繁截屏）
- 历史记录：最近10条

**活动类型识别：**
- 编程
- 文档编辑
- 会议
- 浏览网页
- 休息
- 其他

**截图保存位置：**
- macOS: `~/Library/Application Support/com.workplace.meow/screenshots/`
- Windows: `%APPDATA%/workplace-meow/screenshots/`
- Linux: `~/.local/share/workplace-meow/screenshots/`

### 4. AI 情感分析驱动的 Live2D 表情

**功能描述：**
- 分析 AI 回复的情感
- 根据情感自动切换 Live2D 表情
- 支持多种情感类型
- 使用关键字匹配 + AI 分析双重机制

**实现文件：**
- `src/lib/sentiment-analysis.ts`

**支持的情感类型：**

| 情感 | 触发词 | Live2D 参数 | 表情效果 |
|------|---------|--------------|----------|
| positive | 开心, 棒, 好, 赞, 成功 | 嘴巴张开, 眼睛放大 | 微笑表情 |
| excited | 太棒了, 超级, 激动, 哇 | 嘴巴更大, 眼睛全开 | 兴奋表情 |
| negative | 不好, 糟糕, 担心, 累 | 嘴巴闭合, 眼睛半闭 | 担忧表情 |
| sad | 难过, 伤心, 失望, 遗憾 | 嘴角下弯, 眉毛下压 | 难过表情 |
| angry | 生气, 愤怒, 讨厌 | 眼睛圆睁, 眉毛紧锁 | 愤怒表情 |
| surprised | 天哪, 什么, 不会吧 | 嘴巴张开, 眉毛上扬 | 惊讶表情 |
| neutral | 好的, 明白, 知道了 | 默认表情 | 中性表情 |

**Live2D 参数控制：**
```typescript
// 控制参数示例
'ParamMouthForm': 1,      // 嘴巴形状
'ParamEyeLOpen': 0.8,     // 左眼开合
'ParamEyeROpen': 0.8,     // 右眼开合
'ParamBrowLY': 0.3,       // 左眉毛位置
'ParamBrowRY': 0.3,       // 右眉毛位置
```

**双重分析机制：**
1. **关键字匹配** - 快速匹配常见情感词
2. **AI 分析** - 调用配置的 AI 模型进行深度分析

**触发条件：**
- AI 开始生成回复

### 6. 鼠标跟随（眼睛跟踪）

**功能描述：**
- 实时跟随鼠标移动
- 控制眼睛注视方向
- 头部跟随角度计算
- 支持边界检测

**实现文件：**
- `src/hooks/useMouseTracking.ts`

**Live2D 参数：**
```typescript
'ParamMouseX': -1 ~ 1    // 鼠标X轴位置
'ParamMouseY': -1 ~ 1    // 鼠标Y轴位置
'ParamAngleX': -1 ~ 1    // 头部水平角度
'ParamAngleY': -1 ~ 1    // 头部垂直角度
```

**计算逻辑：**
1. 获取鼠标在窗口中的相对位置
2. 计算归一化的 X/Y 坐标（-1 到 1）
3. 反转 X 轴以匹配直觉
4. 映射到 Live2D 参数范围

**适配功能：**
- DPI 缩放适配
- 窗口大小变化适配
- macOS 适配

### 7. 主动关怀与鼓励

**功能描述：**
- 每30分钟自动鼓励
- 根据场景选择合适的鼓励语
- AI 生成个性化鼓励消息
- 支持多种场景识别

**实现文件：**
- `src/hooks/useActivityMonitoring.ts`

**鼓励场景：**

| 场景 | 时间/状态 | 鼓励语示例 |
|------|----------|----------|
| working_long | 连续工作2小时 | "你已经工作超过2小时了，站起来走动一下吧～" |
| high_productivity | 检测到高效工作 | "哇！今天的效率真高，为你点赞！👍" |
| idle | 休息状态 | "放松一下也好，为下一个冲刺做准备！" |
| night_work | 深夜工作（0-6点）| "这么晚还在工作，要注意休息哦～" |

**AI 驱动：**
- 调用配置的 AI 模型生成个性化鼓励
- 支持所有 AI 提供商
- 无 API 密钥时使用预设消息

**时间配置：**
- 鼓励间隔：30分钟
- 气泡显示时长：8秒

---

### 8. 主动发起对话

**功能描述：**
- 基于当前工作状态主动发起对话
- AI 生成关心的询问
- 提供帮助建议
- 显示气泡15秒

**实现文件：**
- `src/hooks/useActivityMonitoring.ts`

**触发机制：**
1. 结合截屏分析结果
2. 生成个性化关心语
3. 提供针对性帮助

**对话示例：**
```
场景：正在编程
AI回复："注意到你在专注编程，代码还顺利吗？需要我帮你review吗？"

场景：正在写文档
AI回复："看你认真写文档的样子，需要帮你润色一下语言吗？"
```

## 隐私说明

**数据存储：**
- 所有设置存储在本地应用数据目录
- 不会上传到任何服务器
- 截图保存在本地，仅用于本地分析

**截图用途：**
- 仅用于分析当前工作活动
- 不传输到云端
- 可在设置中禁用（待添加开关）

**剪贴板监听：**
- 仅读取剪贴板内容
- 不修改剪贴板
- 内容不会保存或上传

## 技术实现细节

### 前端架构
```typescript
// 系统监控Hook
useSystemMonitor()
  ├── 剪贴板监听
  ├── 番茄钟
  └── 工作会话管理

useActivityMonitoring()
  ├── 自动截屏
  ├── 活动分析
  ├── 鼓励系统
  └── 主动对话

useMouseTracking()
  ├── 鼠标跟随
  ├── 打字动画
  └── 参数控制
```

### 后端架构（Rust）
```rust
// Tauri Commands
capture_screenshot()  // 跨平台截图
get_settings()       // 读取设置
save_settings()      // 保存设置
```

### AI 调用流程
```mermaid
User Action → Detect Trigger → Load AI Config → Call AI Service → Get Response → Update UI → Update Live2D
```


## 故障排除

### 截图失败
**问题：** macOS 提示需要屏幕录制权限

**解决：**
1. 打开系统偏好设置 > 安全性与隐私 > 屏幕录制
2. 允许职场桌宠访问
3. 重启应用

### AI 分析失败
**问题：** 提示"生成回复时出错了"

**解决：**
1. 检查网络连接
2. 验证 API 密钥是否正确
3. 确认所选模型是否支持
4. 查看浏览器控制台错误日志

### 情感表情不工作
**问题：** Live2D 表情没有变化

**解决：**
1. 确认模型支持表情参数
2. 检查模型参数名称是否匹配
3. 查看 Live2D Manager 日志
